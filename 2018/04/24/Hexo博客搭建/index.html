发表于 2018-04-24 | 分类于 Blog | | 阅读次数:

Hexo和WordPress的区别：

对WP大家肯定是不会陌生的，WP作为老牌的博客框架一直深受大家的喜爱。

但是太过臃肿，以前必须要在服务器上才能搭建（又是一笔money），让我们显得不够优雅（逼格太低）

相比之下，hexo足够简单，而且界面漂亮、简洁。

唯一的缺点就是文档太少，遇到的问题一般需要自己去摸索。

WP的优点：

  • 方便部署，上传——访问——设置账号密码，就可以直接进后台开始写了。
  • 方便维护：你所遇到的任何问题，都有人遇到过了，直接搜索问题即可。
  • 及时更新：wordpress是运行多年的项目了，不像很多博客系统，停更很可怕的。
  • 傻瓜无脑：不像HEXO，wordpress是给普通人用的，不需要懂代码，直接用就好。

MarkDown是什么？

在介绍HEXO之前得先说一下MarkDown.因为无论你用什么框架搭建Blog ，写博客的时候还是需要MarkDown 语法的，（就像写字，无论你用楷体还是宋体，前提都是必须先会汉字）

Markdown 是一个格式化语法结构，其次他是一个使用Perl语言编写的用来实现Markdown至HTML转换的工具，它的设计目标就是让写作者尽可能使用容易阅读的语法书写，而不必被繁琐的HTML标签结构所纠缠，

MarkDown的优点：
  • 让你专注于文字而不是排版。
  • 纯文本，所以兼容性极强，可以用所有文本编辑器打开。
  • 格式转换方便，Markdown 的文本你可以轻松转换为 html、电子书等。
  • Markdown 的标记语法有极好的可读性。

HEXO博客搭建步骤

1.1 准备工作： 注册GitHub

注册步骤就不细说了，这都不会还是洗洗睡吧。

注册之后新建一个Public 仓库。

注意点：

因为我们是使用 Github Pages 去搭建我们的静态博客，所以仓库的名称应该是你的用户名+.github.io。这个是一定不能出错的。因为之后你将要访问的你的博客地址就是： https://username.github.io 。

1.2 安装环境配置：

  • 安装Git (可直接下载 Xcode 或者使用 Homebrew进行安装)

2.1 安装Hexo:

  • 终端执行如下命令

    sudo npm install -g hexo

  • 初始化，终端cd到一个指定的目录（我在~/Documents/目录下）执行如下命令

    hexo init <folder>

  • 终端cd到目录下，安装npm

    npm install

  • 此时开启hexo服务就可以在本地（本地预览地址http://localhost:4000 ）预览博客主页了

    hexo s

2.2 关联GitHub仓库，部署到GitHub托管

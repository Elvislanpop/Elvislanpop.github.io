MarkDown是什么？

在介绍HEXO之前得先说一下MarkDown.因为无论你用什么框架搭建Blog ，写博客的时候还是需要MarkDown 语法的，（就像写字，无论你用楷体还是宋体，前提都是必须先会汉字）

Markdown 是一个格式化语法结构，其次他是一个使用Perl语言编写的用来实现Markdown至HTML转换的工具，它的设计目标就是让写作者尽可能使用容易阅读的语法书写，而不必被繁琐的HTML标签结构所纠缠，

MarkDown的优点：
  • 让你专注于文字而不是排版。
  • 纯文本，所以兼容性极强，可以用所有文本编辑器打开。
  • 格式转换方便，Markdown 的文本你可以轻松转换为 html、电子书等。
  • Markdown 的标记语法有极好的可读性。

HEXO的优点

  • 无需主机，无需备案，无需域名

  • 全部生成静态文件，不仅对服务器负载小，而且快

  • 界面简单，流畅优雅 自动部署

1.2 安装环境配置：

  • 安装Git (可直接下载 Xcode 或者使用 Homebrew进行安装)

2.1 安装Hexo:

  • 终端执行如下命令

    sudo npm install -g hexo

  • 初始化，终端cd到一个指定的目录（我在~/Documents/目录下）执行如下命令

    hexo init <folder>

  • 终端cd到目录下，安装npm

    npm install

  • 此时开启hexo服务就可以在本地（本地预览地址http://localhost:4000 ）预览博客主页了

    hexo s

2.2 关联GitHub仓库，部署到GitHub托管

  • 终端 cd 到刚才安装HEXO的根目录下，会有一个 _config.yml的文件，这个就是博客的配置文件。

2.3 HEXO的基本使用命令

以下所有命令必须在博客的根目录下使用

  • 生成新的文件： hexo g
  • 开启HEXO的本地服务 hexo s
  • 清除本地缓存 hexo clean
  • 部署到GitHub hexo d
  • 新建一篇文章 hexo new [layout] <title>
